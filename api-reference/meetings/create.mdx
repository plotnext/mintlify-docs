## Overview

Create or update a meeting in the system. This endpoint automatically creates a bot session for the meeting and schedules it to start 5 minutes before the scheduled time. Supports upsert behavior based on `external_meeting_id`.

## Authentication

<Warning>
This endpoint requires authentication via JWT token with `enterpriseId` and `defaultOrgUnitId` claims.
</Warning>

Provide authentication via:
- **Authorization header**: `Authorization: Bearer YOUR_TOKEN` (required)

## Request Body

<ParamField body="scheduled_start_at" type="string" required>
  ISO 8601 timestamp for when the meeting is scheduled to start
  
  **Example**: `"2024-03-20T14:00:00Z"`
</ParamField>

<ParamField body="scheduled_end_at" type="string" required>
  ISO 8601 timestamp for when the meeting is scheduled to end
  
  **Example**: `"2024-03-20T15:00:00Z"`
</ParamField>

<ParamField body="title" type="string">
  Meeting title/subject
  
  **Default**: `"empty title"`
  
  **Example**: `"Q1 Planning Review"`
</ParamField>

<ParamField body="channel" type="string">
  Meeting platform/channel
  
  **Options**: `"gmeet"`, `"zoom"`, `"teams"`, `"offline"`
  
  **Example**: `"gmeet"`
</ParamField>

<ParamField body="join_url" type="string">
  URL to join the meeting
  
  **Example**: `"https://meet.google.com/abc-defg-hij"`
</ParamField>

<ParamField body="status" type="string">
  Meeting status
  
  **Options**: `"scheduled"`, `"skip"`, `"in_progress"`, `"cancelled"`, `"completed"`, `"deleted"`, `"error"`
  
  **Default**: `"scheduled"`
</ParamField>

<ParamField body="host_user_id" type="string">
  ID of the user hosting the meeting (nullable)
  
  **Example**: `"user-123"`
</ParamField>

<ParamField body="external_meeting_id" type="string">
  External meeting identifier (e.g., from Google Calendar, Zoom). Used for upsert behavior.
  
  **Example**: `"google-cal-event-abc123"`
</ParamField>

<ParamField body="metadata" type="object">
  Additional metadata as a JSON object
  
  **Example**: `{ "calendarId": "primary", "recurringEventId": "abc" }`
</ParamField>

<ParamField body="participant_emails" type="array">
  Array of participant email addresses
  
  **Example**: `["alice@example.com", "bob@example.com"]`
</ParamField>

<ParamField body="recurring_meeting_id" type="string">
  ID linking this meeting to a recurring meeting series
  
  **Example**: `"recurring-series-123"`
</ParamField>

<ParamField body="type" type="string">
  Meeting type
  
  **Options**: `"internal"`, `"external"`
  
  **Default**: `"internal"`
</ParamField>

## Upsert Behavior

The endpoint supports upsert (update or insert) based on `external_meeting_id`:

### If `external_meeting_id` is provided and exists:
- **Updates** the existing meeting
- Preserves `host_user_id` if not provided in request
- Updates `updated_at` timestamp

### If `external_meeting_id` is new or not provided:
- **Creates** a new meeting
- Generates a new meeting ID

<Note>
Use `external_meeting_id` when syncing with external calendar systems (Google Calendar, Outlook, etc.) to prevent duplicate meetings.
</Note>

## Automated Bot Integration

When a meeting is created with status `"scheduled"`:

<Steps>
<Step title="Bot Session Created">
  A bot session record is automatically created with state `"pending"`
</Step>

<Step title="Job Scheduled">
  A background job is scheduled to start **5 minutes before** the meeting's `scheduled_start_at` time
</Step>

<Step title="Bot Joins Meeting">
  At the scheduled time, the bot automatically joins the meeting to record it
</Step>

<Step title="Recording & Transcription">
  Bot records the meeting and generates transcripts
</Step>
</Steps>

### Bot Session States

- `pending`: Bot scheduled, waiting for meeting start
- `active`: Bot has joined and is recording
- `completed`: Recording finished successfully
- `cancelled`: Meeting cancelled, bot won't join
- `error`: Bot encountered an error

## Meeting Cancellation

When creating a meeting with status `"cancelled"` or `"deleted"`:

1. **Existing bot session** is updated to `"cancelled"` state
2. **Scheduled job** is deleted from the job queue
3. **No participants** are created
4. **Bot will not join** the meeting

## Participant Management

### Participant Creation

When `participant_emails` array is provided:

<Check>
**Automatic user lookup**: System checks if email belongs to a registered user
</Check>

<Check>
**Duplicate prevention**: `ON CONFLICT DO NOTHING` prevents duplicate participants
</Check>

<Check>
**Partial failure handling**: If one participant fails, others still process
</Check>

### Participant Fields

- `user_id`: Set if email matches a registered user, otherwise `null`
- `email_id`: Normalized email (lowercased, trimmed)
- `display_name`: Currently `null`, reserved for future use
- `contact_number`: Currently `null`, reserved for future use

## Job Scheduling

Meetings are automatically scheduled in the jobs system:

```json
{
  "name": "meeting.start",
  "options": {
    "singletonKey": "meeting-{MEETING_ID}",
    "startAfter": "{SCHEDULED_START_AT - 5 minutes}",
    "expireInSeconds": 3600,
    "retryLimit": 3,
    "retryBackoff": true,
    "priority": 10
  }
}
```

<Tip>
The `singletonKey` ensures only one job exists per meeting, automatically handling updates to rescheduled meetings.
</Tip>

## Environment Variables

Required environment variables for full functionality:

- `DATABASE_URL`: PostgreSQL connection string
- `JWT_SECRET`: Secret for JWT token verification
- `JOBS_URL`: URL for background jobs API
- `JOBS_AUTH_TOKEN`: Authentication token for jobs API
